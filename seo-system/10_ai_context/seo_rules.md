# SEO Rules

## Core Rules

- Public-facing content is Russian by default unless English is explicitly requested.
- Prioritize Yandex, Google Russia/CIS and local SEO.
- Match the real search intent before writing.
- Use one primary target keyword per article.
- Cover semantic entities naturally.
- Do not keyword-stuff.
- Build topic clusters around real Sever Studio services.
- Add internal links that help the reader choose the next step.
- Include FAQ sections when useful.
- Write metadata for every article.
- Use original structure and Sever Studio facts.
- Mark missing information as `TODO`.

## Every SEO Article Must Include

- search intent
- target keyword
- semantic entities
- clear H1/H2/H3 structure
- short paragraphs
- FAQ section
- internal linking suggestions
- CTA to Sever Studio
- meta title
- meta description

## Local SEO Rules

- Use city modifiers only when the page genuinely serves the local intent.
- Main local focus: Москва, Московская область, Красногорск, Мытищи.
- Keep NAP consistent after admin confirmation.
- Do not create doorway-like duplicate local pages.
- Add local FAQ only when helpful.
- Link local content to contacts and relevant services.

## GEO / AI-Search Rules

- Include direct definitions.
- Answer common decision questions clearly.
- Use step-by-step summaries where appropriate.
- Add comparison tables when they help.
- Make statements concise and citation-friendly.
- Avoid vague, generic paragraphs.
- Do not overpromise.
- Do not add fake proof.

## Internal Linking Rules

- Link from cluster articles to pillar pages.
- Link from pillar pages to service pages.
- Link from service-related articles to relevant commercial pages.
- Use varied, natural Russian anchor text.
- Do not force links.
- For unknown URLs, write `TODO URL`.
