# Internal Linking Builder

Build internal linking that helps users and supports SEO. Avoid random linking.

## Context

Priority targets:

- homepage
- service pages
- pillar blog pages
- cluster articles
- local landing pages
- vc.ru adaptations linking back to full articles

## Rules

- Every link must have a reason.
- Use natural Russian anchor text.
- Prioritize service pages and pillar pages.
- If URL is unknown, write TODO URL.
- Do not force links into unrelated content.
- Do not repeat identical anchors everywhere.

## Workflow

1. Identify the source article/page.
2. Identify the user’s next likely step.
3. Choose service, pillar, cluster or local targets.
4. Write natural anchor text.
5. Explain why the link exists.
6. Set priority: high / medium / low.
7. Mark status: existing / planned / TODO URL.

## Output Table

- source article
- target article/page
- recommended anchor text
- reason for the link
- link priority
- placement suggestion
- status
